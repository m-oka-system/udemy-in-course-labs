## 🎯 Mission

Azure ポータルで仮想ネットワークを作成する手順を確認してみましょう。

## 🚀 Challenge

### 仮想ネットワークの作成

1. 左側メニューの [仮想ネットワーク] をクリックします。
2. ページの上部にあるメニューから [作成] をクリックします。

#### 基本情報

3. 以下のとおり入力して [次へ] をクリックします。

| 項目               | 設定値                       |
| ------------------ | ---------------------------- |
| サブスクリプション | 自動選択される               |
| リソースグループ   | 既存のリソースグループを選択 |
| 仮想ネットワーク名 | VNET1                        |
| リージョン         | Japan East                   |

#### セキュリティ

4. 既定値のまま何も変更せず [次へ] をクリックします。

| 項目                                       | 設定値                |
| ------------------------------------------ | --------------------- |
| 仮想ネットワークの暗号化                   | チェックなし (既定値) |
| Azure Bastion を有効にする                 | チェックなし (既定値) |
| Azure Firewall を有効にする                | チェックなし (既定値) |
| Azure DDoS Network Protection を有効にする | チェックなし (既定値) |

> [!NOTE]
> セキュリティタブの設定は主に有料の機能を有効化するためのものです。
> すべて無効のまま進める手順になっていますが、どんな項目があるのかざっと目を通しておくと良いでしょう。

#### アドレス空間

5. 仮想ネットワークに関する設定を以下のとおり入力します。

| 項目                                    | 設定値                |
| --------------------------------------- | --------------------- |
| IP アドレスプールを使用して割り当てます | チェックなし (既定値) |
| 開始アドレス                            | 10.0.0.0              |
| サイズ                                  | /16                   |

6. 既定のサブネット名 `default` をクリックして編集画面を開きます。
7. サブネットに関する設定を以下のとおり入力して [保存] をクリックします。

| 項目                      | 設定値                |
| ------------------------- | --------------------- |
| サブネットの目的          | Default               |
| 名前                      | Subnet1               |
| IPv4 アドレス空間を含める | チェックあり (既定値) |
| IP アドレスの範囲         | 10.0.0.0/16           |
| 開始アドレス              | 10.0.1.0              |
| サイズ                    | /24                   |
| その他の項目              | 既定値のまま          |

8. [レビューと作成] をクリックします。
9. 入力内容に誤りがないか確認し [作成] をクリックします。
10. デプロイが完了したら [リソースに移動する] をクリックします。
11. 以下のメニューを開いてデプロイ結果を確認してみましょう。

- 概要: アドレス空間が `10.0.0.0/16` になっている
- サブネット: `Subnet1` のアドレス空間が `10.0.1.0/24` になっている
