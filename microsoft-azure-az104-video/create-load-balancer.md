### Azure Load Balancer の作成

1. 左側メニューの [ロードバランサー] をクリックします。
2. ページの上部にあるメニューから [作成する] > [Standard Load Balancer] を選択します。

#### 基本

3. 以下のとおり入力して [次へ] をクリックします。

| 項目               | 設定値                       |
| ------------------ | ---------------------------- |
| サブスクリプション | 自動選択される               |
| リソースグループ   | 既存のリソースグループを選択 |
| 名前               | LB1                          |
| リージョン         | Japan East                   |
| SKU                | Standard (既定値)            |
| 種類               | パブリック                   |
| レベル             | 地域 (既定値)                |

#### フロントエンド IP 構成

4. [フロントエンド IP 構成の追加] をクリックします。
5. パブリック IP アドレスの [新規作成] をクリックします。
6. 以下のとおり入力して [保存] をクリックします。

| 項目         | 設定値       |
| ------------ | ------------ |
| 名前         | IP1          |
| その他の項目 | 既定値のまま |

7. フロントエンド IP 構成を以下のとおり入力して [保存] をクリックします。

| 項目         | 設定値       |
| ------------ | ------------ |
| 名前         | frontend     |
| その他の項目 | 既定値のまま |

8. フロントエンド IP 構成が追加されたら [次へ] をクリックします。

#### バックエンドプール

9. [バックエンドプールの追加] をクリックします。
10. 以下のとおり入力して [保存] をクリックします。

| 項目                     | 設定値       |
| ------------------------ | ------------ |
| 名前                     | bepool       |
| 仮想ネットワーク         | VNET1        |
| バックエンドプールの構成 | NIC (既定値) |
| IP 構成 (※)              | VM1 / VM2    |

※ [追加] をクリック > VM1 / VM2 にチェックを入れる > [追加] をクリック

11. バックエンドプールが追加されたら [次へ] をクリックします。

#### インバウンド規則

12. [負荷分散規則の追加] をクリックします。
13. 正常性プローブの [新規作成] をクリックします。
14. 以下のとおり入力して [保存] をクリックします。

| 項目              | 設定値       |
| ----------------- | ------------ |
| 名前              | probe        |
| プロトコル        | TCP (既定値) |
| ポート            | 80 (既定値)  |
| サイクル間隔 (秒) | 5 (既定値)   |

15. 負荷分散規則を以下のとおり入力して [保存] をクリックします。

| 項目                                                  | 設定値                                         |
| ----------------------------------------------------- | ---------------------------------------------- |
| 名前                                                  | rule                                           |
| IP バージョン                                         | IPv4 (既定値)                                  |
| フロントエンド IP アドレス                            | frontend                                       |
| バックエンドプール                                    | bepool                                         |
| プロトコル                                            | TCP (既定値)                                   |
| ポート                                                | 80                                             |
| バックエンドポート                                    | 80                                             |
| セッション永続化                                      | なし (既定値)                                  |
| アイドルタイムアウト (分)                             | 4 (既定値)                                     |
| TCP リセットの有効化                                  | チェックなし (既定値)                          |
| フローティング IP の有効化                            | チェックなし (既定値)                          |
| アウトバウンドの送信元ネットワークアドレス変換 (SNAT) | (推奨) アウトバウンド規則を使用して〜 (既定値) |

16. [確認および作成] をクリックします。
17. 入力内容に誤りがないか確認し [作成] をクリックします。
18. デプロイが完了したら [リソースに移動する] をクリックします。
19. 以下のメニューを開いてデプロイ結果を確認してみましょう。

- フロントエンド IP 構成
- バックエンドプール
- 正常性プローブ
- 負荷分散規則
